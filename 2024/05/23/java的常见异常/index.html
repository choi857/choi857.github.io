并发修改异常（ConcurrentModificationException）

这玩意我是在并发编程里碰到的。在用迭代器遍历一个集合的时候，同时又有别的线程在修改这个集合，结果就报这个异常了。
并发，死锁之类的也会有新的专门博客来讲这方面（蔡某喜提bug的那些事）

可以用 Iterator 的 remove 方法来安全地删除元素，或者考虑使用 CopyOnWriteArrayList 这种线程安全的集合类。
